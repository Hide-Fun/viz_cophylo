ggbipartiteで始める二部ネットワーク可視化

必要なパッケージ

library(ggplot2)
library(dplyr)

この vignette は、ggbipartite を使った二部ネットワーク可視化を 初心者向けに段階的に解説します。

最終的に、次の 4 パターンを自分のデータで再現できる状態を目指します。

  1. abundance（量）を反映した基本可視化
  2. メタデータに基づく色分け
  3. binary（有無）に特化した可視化
  4. 系統樹とネットワークの対応づけ

まず系統樹を可視化して全体像を掴む

この章で使う主な関数:

  • rtree(): 例示用のランダム系統樹を生成します。
  • ggtree(): 系統樹を ggplot オブジェクトとして描画します。
  • geom_tiplab(): tip ラベルを表示します。
  • patchwork::plot_layout(): 2つの tree を横並びで表示します。

最初に host_phylo と symbiont_phylo を単体で描くと、 「左右どちらがどの生物群か」をイメージしやすくなります。

相互作用行列を作り、系統樹の順序に合わせる

この章で使う主な関数:

  • get_tip_order(): tree 上の y 順に tip ラベル順を取得します。
  • base::factor(levels = ...): host / symbiont の表示順を固定します。
  • arrange() と relocate(): 行列化前の表を順序どおりに並べます。
  • column_to_rownames(): host 列を行名へ変換して行列化します。

ggbipartite では、行列の行順・列順がレイアウトに直接影響します。
そのため、系統樹と対応させる場合は以下の 2 点が重要です。

  1. factor(levels = ...) で表示順を明示する
  2. 行列をその順序でソートしてから可視化に使う
# 系統樹の tip 順と行列の行列順を明示的にそろえる
host_order <- get_tip_order(host_phylo) |> rev()

long 形式へ変換し row, column, count を理解する

この章で使う主な関数:

  • to_longer(): 行列を long 形式へ変換します（0 は除外）。
  • rename() と mutate(): count 列名と因子順を整えます。
  • ggbipartite の主要 geoms は row, column, count を必須入力として使います。

ggplot() に渡すときは、最低限 row, column, count を使います。

  1. row: 左側ノードID（この vignette では host）
  2. column: 右側ノードID（この vignette では symbiont/otu）
  3. count: 相互作用の強さ（カウント、重み）
interaction_df <- to_longer(interaction_matrix, .rowname = "row") |>
  rename(count = interaction) |>
  mutate(
    row = factor(row, levels = rownames(interaction_matrix)),
    column = factor(column, levels = colnames(interaction_matrix))
  )

glimpse(interaction_df)

原理的には、行列と系統樹の左右対応が保たれていれば row / column を逆に定義しても描画は可能です。
ただし実務では混乱を避けるため、最初に 「row をどちら側とみなすか」を固定することを推奨します。

abundance の基本可視化

この章で使う主な関数:

  • geom_bipnet_box(): row / column 側のボックスを描きます。
  • geom_bipnet_interaction(): 相互作用ポリゴンを描きます。
  • stat_bipnet(): geom_bipnet_*() から内部的に呼ばれ、座標計算を行います。
gap_value <- sum(interaction_matrix) / 12

p_abundance <- ggplot(
  interaction_df,
  aes(row = row, column = column, count = count)
) +
  geom_bipnet_box(
    type = "row",
    gap = gap_value,
    fill = "grey85",
    colour = "grey30",
    linewidth = 0.2
  ) +
  geom_bipnet_box(
    type = "column",
    gap = gap_value,
    fill = "#2C3E50",
    colour = "grey10",
    linewidth = 0.2
  ) +
  geom_bipnet_interaction(
    gap = gap_value,
    alpha = 0.55,
    linewidth = 0.15,
    show.legend = FALSE
  ) +
  theme_void()

p_abundance

gap を未指定にすると stat_bipnet() が自動で値を選びます （sum(count) / 10、非正の場合は 1）。
再現性の観点では、gap は明示指定するのが安全です。

補足: geom_bipnet_interaction() では type = "interaction" が既定なので、 通常は省略できます（上のコードも省略形です）。

メタデータで色分けする

この章で使う主な関数:

  • geom_bipnet_box(): row / column のメタデータ色分けに使います。
  • geom_bipnet_interaction(): 連結部もメタデータで色分けできます。
  • geom_bipnet_point(): binary 表示時のノード色分けに使えます。
  • after_stat(): stat_bipnet() が後段で付与する列を参照します。

row 側だけでなく column 側でも同様に色分けできます。
さらに同じ考え方で geom_bipnet_point() と geom_bipnet_interaction() でも色分け可能です。

after_stat() を使う理由は、metadata_row / metadata_column の列が stat_bipnet() の計算後にデータへ結合されるためです。 つまり元データ interaction_df には存在しない列を 審美属性に使うには after_stat() が必要になります。

p_metadata <- ggplot(
  interaction_df,
  aes(row = row, column = column, count = count)
) +
  geom_bipnet_box(
    aes(fill = after_stat(family)),
    type = "row",
    row_nm = "host",
    metadata_row = metadata_row,
    gap = gap_value,
    linewidth = 0.2
  ) +
  geom_bipnet_box(
    aes(fill = after_stat(guild)),
    type = "column",
    column_nm = "symbiont",
    metadata_column = metadata_column,
    gap = gap_value,
    linewidth = 0.2
  ) +
  geom_bipnet_interaction(
    aes(fill = after_stat(guild)),
    column_nm = "symbiont",
    metadata_column = metadata_column,
    gap = gap_value,
    alpha = 0.35,
    linewidth = 0.1,
    colour = NA,
    show.legend = FALSE
  ) +
  theme_void() +
  labs(fill = "Row/Column metadata")

p_metadata

レイアウトパラメータを調整する

この章で使う主な関数:

  • geom_bipnet_box(): gap, box_ratio, ratio, adjust_box_height を渡せます。
  • geom_bipnet_interaction(): 同じレイアウト引数を共有し、各レイヤーの整合を保ちます。
  • stat_bipnet(): 指定パラメータに基づいて座標を再計算します。

box_ratio, ratio, adjust_box_height は次を調整します。

  1. box_ratio: sum(count) / box_ratio でボックス幅を決める。値を大きくするとボックスは細くなる。
  2. ratio: 全体の横方向スケールに影響する。値を小さくすると左右の間隔は広がる傾向がある。
  3. adjust_box_height: TRUE で row 側と column 側の総高さを揃えるよう gap を再調整する。
p_abundance_tuned <- ggplot(
  interaction_df,
  aes(row = row, column = column, count = count)
) +
  geom_bipnet_box(
    type = "row",
    gap = sum(interaction_matrix) / 10,
    box_ratio = 6,
    ratio = 0.75,
    adjust_box_height = TRUE,
    fill = "grey85",
    linewidth = 0.2
  ) +
  geom_bipnet_box(
    type = "column",
    gap = sum(interaction_matrix) / 10,
    box_ratio = 6,
    ratio = 0.75,
    adjust_box_height = TRUE,
    fill = "#2C3E50",
    linewidth = 0.2
  ) +
  geom_bipnet_interaction(
    gap = sum(interaction_matrix) / 10,
    box_ratio = 6,
    ratio = 0.75,
    adjust_box_height = TRUE,
    alpha = 0.5,
    linewidth = 0.15,
    show.legend = FALSE
  ) +
  coord_equal() +
  theme_void()

p_abundance_tuned

binary（有無）表示に切り替える

この章で使う主な関数:

  • to_longer(): binary 行列を long 形式へ変換します。
  • geom_bipnet_interaction(interaction_type = "binary"): 相互作用を線分として描きます。
  • geom_bipnet_point(): row / column ノードを点として描きます。

接続の「強さ」ではなく「存在/非存在」を見たい場合は interaction_type = "binary" を使います。

binary_matrix <- (interaction_matrix > 0) * 1L

binary_df <- to_longer(binary_matrix, .rowname = "row") |>
  rename(count = interaction) |>
  mutate(
    row = factor(row, levels = rownames(binary_matrix)),
    column = factor(column, levels = colnames(binary_matrix))
  )

binary_gap <- sum(binary_matrix) / 10

p_binary <- ggplot(
  binary_df,
  aes(row = row, column = column, count = count)
) +
  geom_bipnet_interaction(
    interaction_type = "binary",
    gap = binary_gap,
    linewidth = 0.45,
    alpha = 0.7,
    show.legend = FALSE
  ) +
  geom_bipnet_point(
    type = "row",
    gap = binary_gap,
    fill = "steelblue",
    size = 2.4
  ) +
  geom_bipnet_point(
    type = "column",
    gap = binary_gap,
    fill = "navy",
    size = 2.4
  ) +
  coord_equal() +
  theme_void()

construct_bn_coordination() でラベル注釈する

この章で使う主な関数:

  • construct_bn_coordination(): row_box / column_box / interaction 座標を明示的に作成します。
  • compute_box_label_coords(): ボックス外側ラベルの代表座標を算出します。
  • annotate(): 算出座標を使ってテキスト注釈を描きます。

construct_bn_coordination() は通常、 geom_bipnet_*() が内部で呼び出している座標計算関数です。
ふだんは直接呼ぶ必要はありませんが、 ラベル座標や外部図形（リンク線など）を自作するときには有用です。

# 各 geom が使う内部座標を一度明示的に計算して再利用する
bn_coords <- construct_bn_coordination(
  .mat = interaction_matrix,
  .row = "host",
  .column = "symbiont",
  .metadata_row = metadata_row,
  .metadata_column = metadata_column,
  .gap = gap_value
)

label_row <- compute_box_label_coords(
  .box = bn_coords$row_box,
  .by = "row",
  .side = "row"
)

label_column <- compute_box_label_coords(
  .box = bn_coords$column_box,
  .by = "column",
  .side = "column"
)

x_span <- diff(range(c(label_row$x, label_column$x)))
label_offset <- x_span * 0.03

ggplot(
  interaction_df,
  aes(row = row, column = column, count = count)
) +
  geom_bipnet_box(
    type = "row",
    gap = gap_value,
    fill = "grey85",
    linewidth = 0.2
  ) +
  geom_bipnet_box(
    type = "column",
    gap = gap_value,
    fill = "#2C3E50",
    linewidth = 0.2
  ) +
  geom_bipnet_interaction(
    gap = gap_value,
    alpha = 0.5,
    linewidth = 0.15,
    show.legend = FALSE
  ) +
  annotate(
    geom = "text",
    x = label_row$x - label_offset,
    y = label_row$y,
    label = label_row$row,
    hjust = 1,
    size = 3
  ) +
  annotate(
    geom = "text",
    x = label_column$x + label_offset,
    y = label_column$y,
    label = label_column$column,
    hjust = 0,
    size = 3
  ) +
  coord_equal(clip = "off") +
  theme_void()

系統樹とネットワークを対応づける

この章で使う主な関数:

  • adjust_tree(): tree の y スケールをボックスに合わせます。
  • create_link(): ボックス中心と tip を結ぶリンク座標を作ります。
  • get_yrange(): 複数パネルの y 範囲を共通化します。
  • geom_tiplab(): tip ラベル（通常版と align = TRUE 版）を描きます。

ここでは次の 3 関数を連携させます。

  1. adjust_tree(): 系統樹の y スケールをボックスに合わせる

よくあるつまずき

この章で参照する主な関数:

  • get_tip_order(): tree と行列の順序不一致を確認します。
  • after_stat(): metadata 列参照に必要な点を再確認します。
  • geom_bipnet_*(): gap, row_nm, column_nm の指定ミスを切り分けます。
  1. 系統樹の tip ラベルと行列の行名・列名が一致していない
    get_tip_order() の返り値と rownames() / colnames() を照合する。
  2. factor(levels = ...) と行列ソートを行っていない
    順序がズレて tree と network の対応が崩れる。
  3. レイヤーごとに gap が違う
    ボックスと相互作用の位置がズレるため、同じ値を使う。
  4. metadata のキー名が row_nm / column_nm と一致していない
    after_stat() で参照する列が生成されない。

まとめ

この章で再確認する主要関数:

  • to_longer()
  • geom_bipnet_box()
  • geom_bipnet_interaction()
  • geom_bipnet_point()
  • adjust_tree() / create_link()

ggbipartite は、次の順番で使うと安定します。

  1. 先に系統樹を描いて左右の意味を固定する
  2. factor と行列ソートで順序を揃える
  3. row / column / count を明示して基本図を作る
  4. 必要に応じて metadata・binary・系統樹連携へ拡張する
